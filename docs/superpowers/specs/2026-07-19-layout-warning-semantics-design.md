# 页面定位告警语义优化设计

## 背景

真实四页案例 `（2026）沪0115民初36914号-2-外网溯源结果报告.pdf` 的页面定位结果包含两条英文告警：第 1 页只有独立“权利图”且没有正式证书或明细汇总表，第 4 页只有“结果1对比图”且没有有效区域。经逐页核对，这两项都是对无关区域的正确排除；第 2 页明细汇总表和第 3 页网页截图已经成功识别，最终详细发布信息评估为无错误。

当前最终规则把布局阶段的所有自由文本 `warnings` 合并到任务告警，并要求 `warnings.length === 0` 才能判定提取完整。因此，模型对正确排除行为的说明会把“0 项问题”的任务降级为“需人工复核”。

## 目标

- 正确排除独立权利图、图片对比区域、基本信息表等非目标内容时，不生成处理告警。
- 页面确认没有目标区域时返回空 `regions`，但仍可给出较高页面置信度。
- 只有疑似存在目标证据但无法可靠定位、分类、确定边界或确定必需序号时，才生成会触发人工复核的告警。
- 面向用户的模型告警统一使用简洁中文。
- 保留现有严格核验原则，不通过关键词过滤或忽略全部布局告警来提升通过率。

## 方案比较

1. **收紧布局提示词的告警语义（采用）**：明确“正确排除不是异常”，限定真正告警的触发条件，并说明空区域页面的置信度含义。改动集中、符合当前模型输出契约，也不会吞掉真正的模糊证据告警。
2. 在最终规则中忽略所有无区域页面的布局告警：实现简单，但可能放过疑似证书或截图因模糊而未被定位的真实异常。
3. 按英文句子或关键词过滤已知告警：只能处理当前样例，依赖模型措辞和语言，容易误判，不采用。

结构化告警代码与严重级别可以作为后续增强，但本轮没有足够多的真实告警类型来稳定定义完整枚举，因此暂不扩展跨阶段输出契约。

## 设计

### 布局提示词语义

在 `LAYOUT_SYSTEM_PROMPT` 中补充以下约束：

- 页面只包含独立权利图、图片对比区域、封面、报告标题、案件基本信息表或其他明确不属于四类目标区域的内容时，返回空 `regions` 和空 `warnings`。
- 不得仅因为页面没有 `certificate`、`rights_screenshot` 或 `summary_table` 而生成告警，也不得把“没有有效区域”本身写成告警。
- 只有页面看起来可能存在目标区域，但由于清晰度、遮挡、边界、类型或必需序号不确定而无法可靠返回时，才写入 `warnings` 并降低置信度。
- `confidence` 表示对页面定位结果完整性和分类正确性的把握；若能明确确认页面没有目标区域，可以返回高置信度。
- 所有告警必须使用简洁中文，描述需要人工核对的具体目标和原因。

### 最终判定

最终判定逻辑保持不变：任何真正的布局告警仍会使任务进入“需人工复核”。本轮不在最终规则中做自由文本过滤，避免把真实漏识别或低置信度情况静默判为通过。

### 已有任务

浏览器历史中的旧任务保存的是处理时的结果，不自动改写。代码部署后需要点击“重新处理”，新提示词才会生效。

## 测试与验收

- 提示词契约测试应断言：正确排除无效内容时不得生成告警。
- 提示词契约测试应断言：空区域页面在确认排除正确时允许高置信度。
- 提示词契约测试应断言：真正不确定的目标区域必须使用中文告警并降低置信度。
- 现有布局 schema、API、流水线和最终规则测试保持通过。
- `npm test`、`npm run typecheck`、`npm run lint` 和生产构建通过。
- 部署后用上述四页真实 PDF 重新处理，确认第 1、4 页不再产生无害告警；若其余证据完整且没有其他复核原因，结果应从“需人工复核 · 0 项”变为“核验通过 · 0 项”。

## 非目标

- 不把独立权利图或图片对比区域改标为 `rights_screenshot`。
- 不把案件基本信息表改标为 `summary_table`。
- 不放宽证书、网页截图、地址栏或汇总表的区域结构校验。
- 不自动修改历史任务结果。
- 不在本轮引入结构化告警代码或跨阶段告警严重级别。
