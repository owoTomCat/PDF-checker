# 历史任务批量管理与三路并发设计

## 背景

当前任务历史只保存在浏览器 `localStorage` 的 `pdf-audit-workspace.tasks.v4` 中，最多 80 条。历史面板只能查看和刷新，上传多个 PDF 后则通过串行 `for` 循环逐个处理，因此同一时间只有一个模型任务运行。

## 目标

1. 支持按任务名称和创建时间筛选历史任务。
2. 支持单条删除、勾选和批量删除。
3. 同一上传批次最多并行处理 3 个 PDF，超过 3 个的任务保持排队，任一任务完成后立即补位。
4. 保持现有任务数据结构、报告查看方式和 `localStorage` 存储键兼容。

## 非目标

- 不增加服务端数据库、跨设备同步或账号级历史记录。
- 不新增正在运行任务的取消能力。
- 不修改百炼阶段接口、PDF 审计规则或最终报告结构。

## 历史管理交互

### 搜索与时间筛选

- 任务名称搜索按 `fileName` 做包含匹配，忽略大小写并去除查询文本首尾空格。
- 时间筛选使用任务 `createdAt`，提供：全部、今天、最近 7 天、最近 30 天、自定义日期范围。
- 自定义结束日期包含该日期全天；只有开始日期或只有结束日期时按单边范围筛选。
- 搜索和时间条件同时生效。
- 改变任一筛选条件时清空当前勾选，避免隐藏记录被误删。
- 无匹配结果时显示“没有符合条件的历史任务”，不改变未显示的数据。

### 选择与删除

- 每条非运行中任务显示复选框和单条删除按钮。
- `queued`、`rendering`、`locating`、`recognizing`、`reviewing_urls`、`extracting_table`、`associating`、`finalizing` 状态不可选择或删除。
- “全选当前结果”只选择当前筛选结果中的可删除任务；再次操作可以取消这些结果的选择。
- 批量工具栏显示当前选中数量，选中数量为零时禁用批量删除。
- 单条和批量删除都先显示浏览器确认框，确认文本包含任务名称或记录数量。
- 删除同步更新 React 状态、`localStorage` 和 `fileCacheRef`。
- 删除当前查看任务后自动展示剩余任务的第一条；没有剩余任务时显示空状态。

## 三路并发处理

- 一次选择或拖入的所有合法 PDF 先全部建立 `queued` 历史记录，确保用户立即看到完整批次。
- 使用固定大小为 3 的前端 worker pool。每个 worker 从同一批次索引中领取下一个任务并执行现有 `processTask()`。
- 批次数量小于 3 时只创建所需 worker；大于 3 时最多 3 个任务处于实际处理阶段，其余保持 `queued`。
- 一个任务失败时由现有任务级错误处理写入 `failed`，worker 继续领取下一任务，不中断批次。
- 当前批次完全结束前保持上传入口禁用，避免多个批次相互穿插；页面概览中的“处理中”按实际活动状态实时统计。
- 批次提示显示“最多同时处理 3 个”，处理结束后显示最后一个任务的正常结果提示。

## 状态与持久化

并行回调不能继续通过“重新读取 localStorage 后覆盖全部任务”的方式更新状态。任务更新改为函数式 `setTasks(current => next)`：在当前内存快照中按 ID upsert、排序、截断至 80 条，再把同一 `next` 写入 `localStorage`。删除操作复用同一持久化入口，避免进度回调、筛选和删除发生丢失更新。

历史筛选条件和勾选状态只存在当前页面内存中，不写入 `localStorage`。任务与报告数据结构保持不变，不升级存储键。

## 代码边界

- `lib/client/task-history.ts`：纯函数，负责名称和时间过滤、任务 upsert、删除 ID 集合及日期边界计算。
- `lib/client/concurrency.ts`：提供固定并发上限的通用执行器；应用传入上限 3，并在单项完成后补位。
- `app/AuditConsole.tsx`：维护筛选和选择状态、渲染管理控件、确认删除、把上传任务交给并发执行器。
- `app/globals.css`：筛选工具栏、复选框、删除操作和移动端换行样式。

## 错误与边界处理

- 无效日期不会抛出异常；该侧日期边界视为未设置。
- `localStorage` 读取失败继续使用空历史；写入失败时保留当前页面内存数据并显示“历史记录未能保存到浏览器”。
- 处理中任务即使因为筛选不可见，也仍正常更新。
- 删除不存在的 ID 为幂等操作。
- 并发执行器接收空数组时立即完成，并拒绝小于 1 的并发数。

## 测试与验收

### 自动化测试

- 名称搜索、全部/预设/自定义时间范围及组合筛选。
- upsert 不覆盖其他并行任务、保持创建时间倒序并截断 80 条。
- 删除 ID 集合只移除目标任务。
- UI 接线断言应用并发上限为 3；通用并发执行器在自定义上限下不超限，且单项失败不阻断其他任务。
- UI 源码包含搜索、时间范围、全选、单删、批量删除和并发上限接线。
- 全量 `npm run typecheck`、`npm test` 通过。

### 浏览器验收

1. 构造多条不同名称和日期的本地历史，验证搜索与时间筛选。
2. 验证单条删除、全选当前结果、取消选择和批量删除。
3. 验证运行中任务的选择与删除操作不可用。
4. 一次上传至少 4 个有效 PDF，确认最多 3 个并行运行，第 4 个先排队后自动开始。
5. 刷新页面，确认删除结果与剩余历史记录持久化，且浏览器没有 hydration 或控制台错误。
