# 当前已知问题

更新时间：2026-07-21

本文记录当前版本已经确认、但尚未完全解决的生产限制。它不包含 API Key、原始 PDF 内容、完整模型响应或用户文件名。

## 模型链路

### 1. 百炼多图请求仍可能暂时不可用

真实 PDF 处理过程中，布局定位等多图请求曾收到百炼侧的 `MODEL_UNAVAILABLE`。同一套 API Key、接入地址、工作空间和模型配置可以通过最小模型请求，因此这类错误更接近上游模型的瞬时可用性或复杂请求负载问题，并不等同于密钥配置错误。

当前任务会保留失败状态和有限的安全错误信息，用户可在原始 PDF 的 72 小时保留期内手动重试。系统尚未实现针对 `MODEL_UNAVAILABLE` 的长时间退避、跨任务自动重试或备用模型切换。

### 2. 汇总表提取可能降级为人工复核

真实 10 页案例中，百炼曾在首次响应和一次结构纠正后仍返回不符合 schema 的汇总表结果。当前版本会将仅发生在汇总表阶段的 `INVALID_MODEL_OUTPUT` 安全降级：不采用不可信的表格行，不继续调用无意义的表格关联模型，并将已有截图以“无法自动关联”写入结果，最终结果应为 `needs_review`，而不是把任务直接标记为失败。

降级后自动表格比对为空，必须由人工核对报告中的汇总表与网页截图。该变更包含回归用例，但按照本次发布要求，没有在发布前重新运行测试，也没有重新提交真实 PDF 触发百炼链路；其真实案例表现仍需后续验证。

### 3. 模型结论仍受扫描质量影响

低分辨率、模糊地址栏、复杂拼版、表格跨页或页面区域边界不清时，页面定位、URL 识别和证据关联仍可能不完整。系统会优先输出 `needs_review`，不会把证据不足的结果自动判定为通过，但这不能代替人工复核。

## 数据与权限

### 4. 数据仍只存放在腾讯云服务器本机

任务历史和结果保存在服务器 SQLite，原始 PDF 保存在服务器私有上传目录并在上传 72 小时后清理。目前尚未接入阿里云 OSS，数据库也没有自动异地备份。因此服务器磁盘损坏、实例丢失或误操作仍可能造成历史任务丢失。

发布采用独立 release 目录，不会主动清理 `/var/lib/pdf-checker`；这只能避免正常代码发布覆盖数据，不能替代 OSS、快照或异地备份。

### 5. 当前是无登录的单租户共享模式

服务器配置为 `PDF_AUDIT_REQUIRE_AUTH=false`。任何能够访问站点的人都能查看共享任务历史，并能删除单个或批量任务。该部署只适合由腾讯云安全组、防火墙或可信反向代理限制访问的受信环境，不适合直接作为公开多用户服务。

## 网络与运维

### 6. 公网 IPv6 链路尚不稳定

服务器本机通过 IPv6 访问应用正常，但当前 Windows 到服务器公网 IPv6 的 SSH/HTTP 连接曾出现提前断开。部署和管理目前使用公网 IPv4 `124.221.73.13`。IPv6 问题可能位于腾讯云安全组、系统防火墙、Nginx 监听或客户端网络路径，尚未完成端到端修复。

### 7. 依赖安全告警尚未逐项处理

最近一次服务器 `npm ci` 的审计摘要报告 3 个依赖漏洞（2 个 moderate、1 个 high）。当前没有执行可能引入破坏性升级的 `npm audit fix --force`。需要在单独的依赖升级任务中确认漏洞路径、实际可利用性和兼容升级方案。

## 本次发布的验证边界

本次发布不运行新的单元测试、类型检查、lint 或真实 PDF/百炼业务测试。部署过程只执行生成生产产物所必需的构建，以及版本号、systemd/Nginx 配置、服务存活和本机 HTTP 健康检查。该边界是发布决定的一部分，不代表上述业务链路已经重新验证通过。
